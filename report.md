# 視窗程式設計 (II) 期末專題報告

**專題名稱**：待辦事項管理系統 (TodoManager)  
**開發環境**：C# WinForms + .NET 8.0  
**資料庫**：SQLite  

---

## 一、專題簡介

本專題開發一套桌面待辦事項管理系統，使用者可透過直覺的圖形介面新增、編輯、刪除待辦事項，
並支援優先等級設定、分類管理、截止日期提醒與完成率統計。
所有資料透過 SQLite 本地資料庫進行永久儲存。

---

## 二、使用說明書

### 2.1 安裝與啟動

1. 確認電腦已安裝 **.NET 8.0 SDK**（可至 https://dotnet.microsoft.com 下載）
2. 開啟 Visual Studio 2022，選擇「開啟專案」→ 選取 `TodoManager.csproj`
3. 在功能表選擇「建置 → 建置方案」
4. 按下 **F5** 或點擊「▶ 執行」按鈕啟動程式

---

### 2.2 主畫面介紹

程式啟動後，畫面分為以下區域：

| 區域 | 說明 |
|------|------|
| **頂部藍色標題列** | 顯示系統名稱與當日日期 |
| **統計卡片列** | 即時顯示全部事項數、已完成數、已逾期數、完成率 |
| **工具列** | 包含搜尋框、狀態篩選、分類篩選、操作按鈕 |
| **事項清單** | 以表格顯示所有待辦事項，逾期項目以紅色標示 |
| **底部狀態列** | 顯示目前篩選結果筆數與操作回饋訊息 |

---

### 2.3 新增待辦事項

1. 點擊工具列上的 **「➕ 新增」** 按鈕
2. 在彈出視窗中填寫以下資訊：
   - **標題**（必填）：簡短描述這件事
   - **描述**（選填）：補充詳細說明
   - **優先等級**：選擇「🟢 低 / 🟡 中 / 🔴 高」
   - **分類**：從下拉選單選擇，或在「新分類」欄位輸入自訂分類
   - **截止日期**：勾選「設定截止日期」後使用日期選擇器設定
3. 點擊 **「💾 儲存」** 完成新增

> 💡 提示：按下 Enter 鍵等同於點擊儲存；按 Esc 取消。

---

### 2.4 編輯待辦事項

- 在清單中**單擊選取**一筆事項，再點擊 **「✏️ 編輯」** 按鈕
- 或直接在清單中**雙擊**該事項
- 修改完成後點擊「💾 儲存」

在編輯視窗中，可勾選 **「標記為已完成」** 來完成該事項。

---

### 2.5 刪除待辦事項

- **單筆刪除**：選取事項後點擊 **「🗑️ 刪除」**，確認後刪除
- **批次清除**：點擊 **「🧹 清除所有已完成事項」**，一次刪除全部完成項目

---

### 2.6 切換完成狀態

選取一筆事項，點擊 **「✅ 切換完成」**：
- 若為「進行中」→ 標記為「已完成」（清單顯示灰色）
- 若為「已完成」→ 恢復為「進行中」

---

### 2.7 搜尋與篩選

工具列提供三種篩選方式，可同時組合使用：

| 篩選器 | 說明 |
|--------|------|
| 🔍 搜尋框 | 輸入關鍵字，即時搜尋標題與描述 |
| 狀態篩選 | 選擇「全部 / 進行中 / 已完成 / 已逾期」 |
| 分類篩選 | 依自訂分類過濾 |

---

### 2.8 逾期提醒

若事項設有截止日期且尚未完成，且截止日期早於今日，該事項會以 **紅色背景** 顯示，
統計卡片中的「⚠️ 已逾期」數字也會更新。

---

## 三、系統架構

```
資料流：使用者操作 → MainForm/TodoEditForm → DatabaseHelper → SQLite (.db)
```

| 元件 | 職責 |
|------|------|
| `TodoItem.cs` | 資料模型，定義所有欄位與輔助屬性 |
| `DatabaseHelper.cs` | 封裝 SQLite CRUD，自動建立資料庫與資料表 |
| `MainForm.cs` | 主視窗，負責列表顯示、統計更新、工具列操作 |
| `TodoEditForm.cs` | 新增／編輯對話框，包含輸入驗證 |
